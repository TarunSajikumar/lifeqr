ALGORITHM 1: Patient Registration Module
Total Lines: 30

ALGORITHM: PatientRegistration
INPUT:  name, email, password, role, age, bloodGroup, healthIssues,
        allergies, medications, emergencyContactName,
        emergencyContactPhone, emergencyContactRelationship,
        phone, address, city, state
OUTPUT: JWT token, user object, unique QR Code ID
STEP 01: START
STEP 02: RECEIVE registration form data from HTTP POST /api/auth/register
STEP 03: VALIDATE required fields: name, email, password, role
STEP 04: IF any required field is missing THEN
STEP 05:     RETURN HTTP 400 error "Required fields missing"
STEP 06: END IF
STEP 07: QUERY MongoDB for existing user with same email
STEP 08: IF duplicate email found THEN
STEP 09:     RETURN HTTP 409 error "Email already registered"
STEP 10: END IF
STEP 11: HASH password using bcrypt with salt rounds = 10
STEP 12: INITIALIZE userData object with: name, email, hashedPassword, role, gender, phone, address
STEP 13: IF role == "patient" THEN
STEP 14:     SET userData.age, bloodGroup, healthIssues, allergies, medications
STEP 15:     IF emergencyContactName AND emergencyContactPhone are provided THEN
STEP 16:         SET userData.emergencyContact = { name, phone, relationship }
STEP 17:     END IF
STEP 18:     CALL generatePatientQrCodeId(name) to produce unique QR ID
STEP 19:     FORMAT QR ID as: [FirstThreeLetters][DateSegment][RandomSuffix]
STEP 20:     VERIFY ID uniqueness by querying MongoDB, regenerate if duplicate
STEP 21:     CONSTRUCT emergency URL: frontendUrl + /emergency_access.html?id=QR_ID
STEP 22:     GENERATE QR code image (PNG, 300x300) using QRCode.toDataURL()
STEP 23:     STORE QR image as Base64 Data URL in userData.qrCode
STEP 24: ELSE IF role == "doctor" THEN
STEP 25:     SET userData.specialization, licenseNumber, hospital
STEP 26: ELSE IF role == "crew" THEN
STEP 27:     SET userData.vehicleNumber, crewType, station
STEP 28: END IF
STEP 29: SAVE user to MongoDB via User.create(userData)
STEP 30: SIGN JWT token with userId and role; RETURN token + user; END

ALGORITHM 2: QR Code Generation Module
Total Lines: 30

ALGORITHM: QRCodeGeneration
INPUT:  patientName, patientId (MongoDB ObjectId), frontendBaseUrl
OUTPUT: Unique QR Code ID string, QR Code PNG image (Base64 Data URL)
STEP 01: START
STEP 02: RECEIVE patient name from registration or profile update trigger
STEP 03: CALL formatNamePrefix(name): strip non-alpha chars, uppercase, take first 3 letters
STEP 04: IF cleaned name is empty THEN SET prefix = "USR" END IF
STEP 05: CALL getDateSegment(): get current day, month, year (2-digit)
STEP 06: RANDOMLY select one of [day, month, year] as the date segment
STEP 07: CALL randomSuffix(length): length is randomly 1 or 2 characters
STEP 08: GENERATE characters from charset: A-Z + 0-9
STEP 09: CONCATENATE: qrCodeId = prefix + dateSegment + suffix
STEP 10: Example output: "SUJ060K" or "SUJ2606A"
STEP 11: QUERY MongoDB: IF User.findOne({ qrCodeId }) exists THEN
STEP 12:     LOOP BACK to STEP 03 and regenerate ID (collision avoidance)
STEP 13: END LOOP (exit when unique ID confirmed)
STEP 14: ASSIGN confirmed unique qrCodeId to userData.qrCodeId
STEP 15: BUILD emergency access URL: frontendUrl + "/emergency_access.html?id=" + qrCodeId
STEP 16: CONFIGURE QRCode options: errorCorrectionLevel = 'H', type = 'image/png'
STEP 17: SET QR image dimensions: width = 300, margin = 2
STEP 18: SET QR color: dark = '#000000', light = '#FFFFFF'
STEP 19: CALL QRCode.toDataURL(emergencyURL, options) to get Base64 PNG string
STEP 20: STORE Base64 QR string in userData.qrCode field
STEP 21: SAVE user document to MongoDB with qrCode and qrCodeId
STEP 22: ON profile update: RECALCULATE QR with color dark = '#6818f4' (purple branded)
STEP 23: OVERWRITE existing user.qrCode with new generated QR
STEP 24: ON /api/patient/regenerate-qr POST request (authenticated patient only)
STEP 25: FETCH patient from DB using JWT userId
STEP 26: IF user not found OR role is not "patient" THEN RETURN 404 error END IF
STEP 27: RE-GENERATE QR code using same qrCodeId (URL remains same)
STEP 28: UPDATE user.qrCode with newly generated Base64 PNG
STEP 29: SAVE updated document to MongoDB
STEP 30: RETURN regenerated QR code as JSON response; END

STEP 10:     RETURN HTTP 404 error "Patient not found"

STEP 20:     CONSTRUCT full response with:
STEP 21:     { qrCodeId, name, gender, age, bloodGroup, allergies, healthIssues,

ALGORITHM 6: JWT Authentication & Backend Database Integration
Total Lines: 30

ALGORITHM: JWTAuthenticationSystem
INPUT:  email, password (for login); token (for protected routes)
OUTPUT: JWT token string (on login); decoded user payload (on verification)
STEP 01: START
STEP 02: LOGIN FLOW:
STEP 03: RECEIVE HTTP POST /api/auth/login with { email, password }
STEP 04: VALIDATE: IF email or password is missing THEN RETURN 400 error END IF
STEP 05: QUERY MongoDB: User.findOne({ email })
STEP 06: IF user not found THEN RETURN HTTP 401 "Invalid email or password" END IF
STEP 07: CALL bcrypt.compare(inputPassword, user.hashedPassword)
STEP 08: IF password does not match THEN RETURN HTTP 401 "Invalid credentials" END IF
STEP 09: CHECK user.active flag: IF active == false THEN RETURN 403 "Account deactivated" END IF
STEP 10: SIGN JWT: jwt.sign({ userId, role }, JWT_SECRET, { expiresIn: '7d' })
STEP 11: RETURN HTTP 200 with { message, token, user: { id, name, email, role, qrCode } }
STEP 12: TOKEN VERIFICATION MIDDLEWARE:
STEP 13: EXTRACT token from request header: Authorization.split(' ')[1]
STEP 14: IF no token THEN RETURN HTTP 401 "No token provided" END IF
STEP 15: CALL jwt.verify(token, JWT_SECRET) to decode payload
STEP 16: IF verification throws error THEN RETURN HTTP 401 "Invalid token" END IF
STEP 17: ATTACH decoded payload to req.user for downstream route handlers
STEP 18: CALL next() to proceed to protected route handler
STEP 19: DATABASE INTEGRATION (MongoDB / Mongoose):
STEP 20: ON server start: CALL mongoose.connect(MONGO_URI, options)
STEP 21: SET options: serverSelectionTimeoutMS=10000, socketTimeoutMS=45000, maxPoolSize=10
STEP 22: IF MONGO_URI starts with "mongodb+srv://" THEN use public DNS (8.8.8.8, 1.1.1.1)
STEP 23: ON successful connection: LOG "MongoDB Connected Successfully"
STEP 24: ON connection failure in production: LOG error and process.exit(1)
STEP 25: ON connection failure in development: START server without DB in limited mode
STEP 26: DEFINE User schema with Mongoose: name, email, password, role, qrCode, qrCodeId,
STEP 27:   age, bloodGroup, allergies, medications, emergencyContact, reports, sosAlerts
STEP 28: USE bcrypt.hash(password, 10) for all stored passwords
STEP 29: HANDLE SIGTERM gracefully: close mongoose connection before exit
STEP 30: RETURN authenticated session with role-based route access; END

ALGORITHM 7: QR Scanner & Live Location Module

ALGORITHM: QRScannerAndLiveLocation
INPUT:  Camera video stream (device camera), lat/lng (GPS coordinates)
OUTPUT: Decoded QR content (patient URL), location stored in DB
STEP 01: START
STEP 02: QR SCANNER FRONTEND LOGIC:
STEP 03: REQUEST camera permissions using navigator.mediaDevices.getUserMedia()
STEP 04: IF permission denied THEN SHOW error: "Camera access required for QR scan" END IF
STEP 05: START video stream and render to video HTML element on page
STEP 06: CREATE canvas element for frame capture and pixel analysis
STEP 07: ON each animation frame: DRAW video frame to canvas using drawImage()
STEP 08: EXTRACT pixel data from canvas using getImageData() method
STEP 09: PASS pixel data to jsQR() or ZXing decoder library for analysis
STEP 10: IF QR code is detected THEN
STEP 11:     EXTRACT decoded URL string from QR result object
STEP 12:     PARSE URL: extract id= query parameter (the patient QR Code ID)
STEP 13:     STOP camera stream (avoid continuous scanning after detection)
STEP 14:     NAVIGATE to emergency_access.html?id=[extractedId]
STEP 15: ELSE
STEP 16:     CONTINUE scanning loop until QR is detected or user cancels
STEP 17: END IF
STEP 18: LIVE LOCATION UPDATE:
STEP 19: ON patient dashboard load: CHECK if Geolocation API is supported
STEP 20: CALL navigator.geolocation.watchPosition() for continuous updates
STEP 21: ON position update: EXTRACT lat, lng from position.coords object
STEP 22: SEND HTTP PUT /api/patient/location with { lat, lng } and JWT Bearer token
STEP 23: VALIDATE: IF typeof lat is not number OR typeof lng is not number THEN RETURN 400 error
STEP 24: QUERY MongoDB: find patient by userId from JWT payload
STEP 25: IF patient not found OR role is not "patient" THEN RETURN 404 error END IF
STEP 26: UPDATE patient.lastLocation = { lat, lng, updatedAt: new Date() }
STEP 27: SAVE updated document to MongoDB
STEP 28: RETURN HTTP 200 with { message: "Location updated", lastLocation }
STEP 29: DISPLAY updated location on patient dashboard map UI
STEP 30: END

ALGORITHM: SOSAlertAndReportUpload
INPUT:  JWT token, lat/lng/message/sentTo (SOS), file/category/description (Report)
OUTPUT: SOS alert record in DB, uploaded report record in DB
STEP 01: START
STEP 02: SOS ALERT FLOW:
STEP 03: RECEIVE HTTP POST /api/patient/sos with Authorization header
STEP 04: AUTHENTICATE JWT token using authenticateToken middleware
STEP 05: EXTRACT { lat, lng, message, sentTo } from request body
STEP 06: VALIDATE: IF typeof lat is not number OR typeof lng is not number THEN
STEP 07:     RETURN HTTP 400 error "Latitude and longitude are required"
STEP 08: END IF
STEP 09: QUERY MongoDB: User.findById(userId from JWT)
STEP 10: IF user not found OR role is not "patient" THEN RETURN 404 error END IF
STEP 11: BUILD sosAlert object:
STEP 12:     { status: 'sent', location: { lat, lng },
STEP 13:       message: message OR 'Emergency SOS request',
STEP 14:       sentTo: (Array.isArray(sentTo) ? sentTo : []),
STEP 15:       createdAt: new Date() }
STEP 16: PREPEND sosAlert to user.sosAlerts array using unshift()
STEP 17: PREPEND SOS activity log to user.activities array:
STEP 18:     { type: 'SOS', title: 'Emergency SOS Sent', description, metadata, timestamp }
STEP 19: SAVE user document to MongoDB with updated sosAlerts and activities
STEP 20: RETURN HTTP 200 { message: "SOS alert created", sosAlert }
STEP 21: MEDICAL REPORT UPLOAD FLOW:
STEP 22: RECEIVE HTTP POST /api/patient/reports/upload with multipart/form-data
STEP 23: AUTHENTICATE JWT and verify role == "patient"
STEP 24: IF no file provided THEN RETURN HTTP 400 "Report file is required" END IF
STEP 25: SAVE uploaded file to /uploads/reports/ folder using multer middleware
STEP 26: BUILD report object:
STEP 27:     { filename, originalName, mimeType, url: '/uploads/[filename]',
STEP 28:       category: category OR 'General', description, uploadedAt: new Date() }
STEP 29: PREPEND report to user.reports array and add "Report Upload" activity log
STEP 30: SAVE to MongoDB; RETURN HTTP 200 { message: "Report uploaded", report }; END
